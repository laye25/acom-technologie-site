# Algorithmic Specification: Adaptive Tatami Fill
**Maturité (Règle 52) :** Implemented  

This document specifies the mathematical formulation for the **Adaptive Tatami Fill Engine** inside the Acom Embroidery Engine (AEE).

---

## 1. Problem Definition

A standard Tatami fill covers a large polygon $\mathcal{P}$ with parallel lines of running stitches. In traditional engines, Tatami is filled with a constant angle $\theta$, constant density $\rho$, and constant stitch length $L_{\text{stitch}}$.

The **Adaptive Tatami Engine** continuously modulates these parameters based on:
1. Local boundary curvature.
2. Local material thickness and elasticity.
3. Density requirements for stability.

---

## 2. Ray-Casting Boundary Intersections

To fill a polygon $\mathcal{P}$, the engine projects parallel rays at angle $\theta$ spaced by the density step $H = 1/\rho$ (typically $0.4\text{ mm}$ to $0.6\text{ mm}$).

Let a ray be parameterized by:

$$\mathbf{R}(t) = \mathbf{O} + t \mathbf{U}$$

Where $\mathbf{U} = (\cos\theta, \sin\theta)$ is the unit direction vector, and $\mathbf{O}$ is the ray origin offset.
For each polygon edge $\mathbf{E}_i$ connecting vertex $\mathbf{V}_i$ to $\mathbf{V}_{i+1}$:

$$\mathbf{E}_i(u) = \mathbf{V}_i + u (\mathbf{V}_{i+1} - \mathbf{V}_i), \quad u \in [0, 1]$$

The intersection point $\mathbf{R}(t_j) = \mathbf{E}_i(u_j)$ is solved via linear systems. The sorted set of intersections $\{t_0, t_1, ..., t_{2k-1}\}$ defines the active spans inside the polygon:

$$\text{Spans} = \bigcup_{j=0}^{k-1} [t_{2j}, t_{2j+1}]$$

---

## 3. Stitch Point Placement & Fractional Row Offsets

Within each span $[t_{2j}, t_{2j+1}]$, stitch points are placed at intervals of $L_{\text{stitch}}$ (typically $3.0\text{ mm}$ to $4.5\text{ mm}$).

To prevent alignment patterns (which form structural lines or "gaps" in the embroidery), the starting offset of each row $r$ is shifted by a fractional coefficient $f_r \in (0, 1)$:

$$\text{Offset}_r = (r \bmod N) \cdot \Delta_{\text{offset}} \cdot L_{\text{stitch}}$$

Where $N$ is the offset pattern repeat count (e.g., $N=4$ for a 1/4 offset pattern) and $\Delta_{\text{offset}} = 0.25$.

---

## 4. Adaptive Variable-Angle Vector Fields

To make Tatami adaptive, instead of a constant angle $\theta$, the engine defines an angle field $\theta(\mathbf{x})$ over the polygon interior.

1. **Laplacian Field Interpolation**: Given boundary angles specified at outer nodes, the internal vector field is computed by solving Laplace's equation:
   
$$\nabla^2 \theta(\mathbf{x}) = 0, \quad \theta(\mathbf{x})\Big|_{\partial \mathcal{P}} = \theta_{\text{boundary}}$$

2. **Streamline Tracking**: Stitch paths are generated by integrating along the streamlines of the vector field $\mathbf{V}(\mathbf{x}) = (\cos\theta(\mathbf{x}), \sin\theta(\mathbf{x}))$ using Runge-Kutta 2nd order (RK2):

$$\mathbf{x}_{k+1} = \mathbf{x}_k + \Delta s \cdot \mathbf{V}\left(\mathbf{x}_k + \frac{\Delta s}{2} \mathbf{V}(\mathbf{x}_k)\right)$$

This mathematical approach allows AEE to sew beautiful, organic curving flows matching the contours of leaves, petals, or custom graphic shapes automatically.
